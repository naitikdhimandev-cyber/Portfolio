**🧠 Problem Statement**

Writing proper documentation is one of the most time-consuming tasks for developers.

Many developers struggle with:

        • Maintaining clear documentation for complex codebases  
        • Explaining logic and architecture in readable format  
        • Keeping documentation updated alongside code  
        • Generating professional developer documentation quickly  


As a result, many projects lack proper documentation, making collaboration and maintenance difficult.



---

**🚀 Solution**

We developed **CodeDoc AI**, an AI-powered system that automatically analyzes source code and generates structured developer documentation.

The system uses AI models to understand:

        • Code structure  
        • Functions and logic  
        • Programming language context  
        • Code behavior  


It then produces readable documentation with formatting, explanations, and structured output suitable for developers and technical teams.

## 🎯 Purpose

---

## 🧩 Detailed Explanation

### Task Class

        • Constructor initializes properties such as id, title, description and priority  
        • The task is initialized with completed status set to false  

        • markComplete()
        -> Marks the task as completed

        • updateTitle(newTitle)
        -> Updates the task title with the provided value


### TaskManager Class

        • Constructor initializes an empty task list  

        • addTask(task)
        -> Adds a new task to the task array

        • getTaskById(id)
        -> Retrieves a task using its id

        • deleteTask(id)
        -> Removes a task using its id

        • getPendingTasks()
        -> Returns all tasks that are not completed



---

**Inputs**

        • Task id

        • Priority  


**Outputs**

**Space Complexity**
